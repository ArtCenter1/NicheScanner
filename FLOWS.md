## 1. 用户主流程

```
1. 打开页面 → 看到 Hero + 输入区（空状态）
2. 在 textarea 输入 1–5 个想法（逗号或换行分隔）
3. 点击 "Analyze" 按钮
4. 看到加载状态（进度指示器，显示当前阶段：scraping → scoring → enriching）
5. 看到结果（排名列表，每个想法有评分和维度分解）
6. （可选）点击 "Download PDF" 下载报告
7. （可选）修改输入，重新分析
```

## 2. 页面 / 入口清单

Niche Scanner MVP 只有 **一个页面**（单页应用），所有交互在同一个视口内完成。

| 页面 / 入口 | 作用 | 从哪里进入 | 下一步 |
|---|---|---|---|
| `首页 (/)` | 输入想法 → 展示结果 → 下载 PDF | 浏览器直接打开 | — |

## 3. 关键交互

### 交互：输入并分析

- **触发**：用户在 textarea 输入 1–5 个想法（以逗号、换行或分号分隔），点击 "Analyze" 按钮
- **即时反馈**：
  - 按钮变为 "Analyzing..." 禁用状态 + 旋转加载图标
  - 输入区域变为只读（不可修改）
  - 下方出现进度卡片，显示当前阶段（Scraping... → Scoring... → Generating recommendations...）
- **结果**：
  - 进度卡片替换为结果区域，显示排名列表
  - 每个想法：总分 + 4 维度 bar chart + 行动建议 + 数据来源标注
  - "Download PDF" 按钮出现
- **禁用条件**：
  - 输入为空时 "Analyze" 按钮置灰
  - 超过 5 个想法时提示 "最多 5 个想法"
  - 单个想法超过 200 字符时提示缩短
  - 正在分析时按钮禁用
- **失败时**：
  - 整体失败 → 显示错误横幅 + "重试"按钮
  - 部分数据失败 → 结果仍显示，对应维度标注 "数据暂缺"

### 交互：下载 PDF 报告

- **触发**：点击 "Download PDF" 按钮
- **即时反馈**：按钮变为 "Generating PDF..." + 旋转图标
- **结果**：浏览器下载 PDF 文件（自动命名 `niche-scanner-report-{date}.pdf`）
- **失败时**：Toast 提示 "PDF 生成失败，请重试"

### 交互：重新分析

- **触发**：分析完成后，用户修改输入并再次点击 "Analyze"
- **即时反馈**：同首次分析（按钮禁用 + 进度指示）
- **结果**：旧结果被新结果覆盖
- **注意**：不支持多结果对比（v1 特性）

## 4. 状态流转

```text
初始（空状态） -- 输入想法 --> 输入就绪
输入就绪 -- 点击 Analyze --> 分析中（scraping）
分析中（scraping） -- 抓取完成 --> 分析中（scoring）
分析中（scoring） -- 评分完成 --> 分析中（enriching）
分析中（enriching） -- LLM 完成 --> 完成（有结果）
分析中 -- 全部失败 --> 完成（错误）
分析中 -- 部分失败 --> 完成（部分结果）
完成 -- 修改输入 --> 输入就绪
```

## 5. 四类状态

| 页面 / 模块 | 加载中 | 空状态 | 错误态 | 成功态 |
|---|---|---|---|---|
| **输入区** | 输入区只读，按钮 "Analyzing..." + spinner | 占位符 "输入你的商业想法，每行一个…"（无分析按钮禁用） | 输入区恢复正常，错误横幅在顶部 | 输入区恢复正常，结果显示在下方 |
| **进度卡片** | 显示当前阶段名称 + 动画进度条 | 隐藏（初始状态） | 隐藏 + 顶部错误横幅 | 隐藏 + 结果区域出现 |
| **结果区域** | Skeleton loading（灰色占位卡片 × 输入数） | 隐藏（无结果） | 顶部错误横幅 + "重试" 按钮 | 排名卡片列表 |
| **PDF 下载按钮** | "Generating PDF..." + spinner | 隐藏 | Toast "生成失败，请重试" | "Download PDF" 按钮可用 |
| **排名卡片** | Skeleton 灰色块 | — | 灰底 + 标注 "XX 数据暂缺"（部分失败） | 完整评分 + bar chart + 建议 |

## 6. 边界和异常

- **输入为空**：Analyze 按钮置灰，无法点击
- **输入超过 5 个**：弹窗/提示 "最多分析 5 个想法"，截断前 5 个
- **单个想法过长（>200 字符）**：红色边框提示 "请缩短到 200 字以内"
- **重复点击 Analyze**：按钮已禁用，无影响
- **网络失败（前端 → 后端）**：红色 Toast "无法连接服务器，请检查网络"
- **抓取全部失败**：结果区域显示 "所有数据源暂不可用，请稍后重试" + 重试按钮
- **抓取部分失败**：对应维度显示灰色 + "XX 数据暂缺"，其他维度正常
- **PDF 生成超时**：Toast "报告生成超时，请重试"
- **后端处理超时（>120s）**：前端超时提示 + "回到首页重新分析" 按钮
- **移动端窄屏**：卡片竖排单列，bar chart 自适应宽度
